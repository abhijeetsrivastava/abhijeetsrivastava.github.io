Dotfiles: Why you need them and how to use them.

October 21, 2021 2-minute read

Every pragmatic programmer has their own preference when it comes to tools, IDE, etc. Nowadays even desktop setup, keyboards etc.

I am going to highlight one such preference of tools and setups.

It’s a great idea to keep a list of popular software you need and any packages for text editors or IDE.

Let’s focus on macos which I am using since last 4 years now. Doing a similar setup in linux is straight forward and could be done in windows too.

Tools one would need

A package manager.

I use macos and thus ‘brew’ is my choice of package manager.

A list of packages and applications one uses.

I have a few favorite packages and brew lets me fetch the package list that I have installed in a BrewFile which I keep updated. This list is great to have so that you do not need to remember your favorite softwares.

Personalized settings for favorite IDE/editors.

  1. I use emacs, vim and sublime as editors. Each has a different usecase.
  2. For VSCode, Intellij I save my settings on the web.
  3. The other most important thing that you might want to backup is your configs. That way you always carry your configs wherever you do.

A storage for above configs

A private repository in github or similar is more than enough to save your configurations.

[Optional] A script for automated restore and backup.

There is a secret hidden step before this one. If you are keeping backup of your config files, you would ideally would want to sym link that when doing a fresh install. The Symlink creation could be automated easily using a restore script.

Conclusion

The steps usually remain the same, just your choice of softwares, package managers would differ. Using this method I have been able to keep a consistent setup between different laptops and also being able to restore my configs relatively faster.

Happy coding!
