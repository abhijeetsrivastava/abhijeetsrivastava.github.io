How to remove null from code?

July 21, 2020 2-minute read

Introduction

Scala: Though the concepts mentioned in this post are related to Scala, they are universal.

Thought: A program should not use null.

Null, if not handled, can be troublesome just like an unhandled exception. One should actively try and avoid its usage.

Here are a couple of ways to avoid them.

How to avoid them?

Without initial default value

What to do when a var field in a class or method doesn’t have an initial default value?

Solution: Initialize it with Option instead of null.

case class student(name: String, address: Option[String])

When a method doesn’t produce the intended result

You may be tempted to return null.

Soulution: Use an Option or Try instead.

def readFile(fileName: String): Option[List[String]] = {
  try {
    Some(io.Source.fromFile(fileName).getLines.toList)
  } catch {
    case e: Exception =>
      Log.info("File does NOT exist")
      None
  }
}

Try/Success/Failure approach

This approach is also useful when using Futures.

object Test {
  def readFile(fileName: String): Try[List[String]] = {
    Try(io.Source.fromFile(fileName).getLines.toList)
  }

  var fileName = "/etc/passwd"
  readFile(fileName) match {
    case Success(lines) => lines.foreach(println)
    case Failure(f) => println(f)
  }
}

The “Null Object” Pattern

Twitter Effective scala page recommends not overusing Option, and using the Null Object Pattern where it makes sense.

A Null Object is an objectthat extends a base type with a “null” or neutral behavior.

trait Animal {
  def makeSound()
}

class Dog extends Animal {
  def makeSound() { println("woof") }
}

class NullAnimal extends Animal {
  def makeSound() {}
}

Legacy Java Code

Solution: wrap it in an Option or Some

def getName: Option[String] = {
  var name = javaPerson.getName
  if (name == null) None else Some(name)
}

Tips

  1. Map of Seq[Option[Object]] skips None values
def print(names: Seq[Option[String]]): Unit = {
  names.map(_.println)
}
  1. Use flatten to convert Seq[Option[Object]] to Seq[Object] Useful for use it with other collections.
def convert(names: Seq[Option[String]]): Seq[String] =
  names.flatten
  1. Use match pattern Similar to Try/Success/Failure
val name: Option[String]
name match {
  Some(name) => println(name)
  None => println("No such name")
}

Conclusion

Null can be easily mishandled and hence we should avoid its possibility in our code. I have listed a few ways to do so.
