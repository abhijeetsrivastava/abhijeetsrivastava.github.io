Programmers best friend IDE

September 13, 2020 7-minute read

A programmer needs a good IDE

A good IDE is a programmers best friend. Writing clean code requires one to spend time in their code base and to have a look and feel of various parts of code. Search files, syntax highlighting, completions, renaming, formatting with fmt flavored styles etc are few of the many things that programmers rely on to write clean code.

Its of paramount importance that one knows their own IDE same as they would know about the car they drive. I mean, one does NOT needs to be know everything about their car to drive it, but then the ones that know everything about their car are the one of the best drivers. Same applies to IDEs as well.

The process one follows from choosing an IDE of their liking starts from early days. We usually use the ones our peers or seniors are using at a project or a company and then we get to pick the flavors/features of IDEs that they rely on.

Things I look for in an IDE

I am a concious programmer. I like to review the code I wrote previously to see if I can refactor it or if its idomatic. In order for me to do so I use certain features of an IDE.

  1. Return types of functions - This is fairly obvious. One needs this to be able to club functions together.
  2. Code duplication - Some IDEs offer code duplication blocks. One should use these blocks to figure out possible refactoring.
  3. Code formatting - Specially useful when a lot of people are working in the same repository and in general its good practice to use a code formatter to enforce idiomatic code styles. I usually try and use the default formatter for a language. If at all, I customize it based on some other popular company (like twitter, airbnb etc).
  4. Jump to definition/ references - One of the most important features for development or debugging a code. One should be able to move seamlessly between references and definitions.
  5. Syntax highlighting and others are useful features which should be easily available.

My IDE journey

My java development started with Eclipse where I also tried Netbeans, BlueJ, DrJava and a few others. Most of these IDEs are for java (DrJava) so I am going to stick to IDEs that are not limited to a particular language.

Eclipse

Best first editor. My reason was it was being used by my peers and seniors and I had help if I needed anything. I was a junior developer and cared less about IDEs and more about startup time of my server. Those were fun days where I had to use slow local setup.

Sublime

During my masters journey, most of my time was spent learning and writing code. I could not be bothered with learning new tools or IDEs. Eclipse is bulky and might not be best for other languages.

My Solution? use a text based editor I was familiar with - Sublime. Sublime has an active community with a lot of useful plugins.

To be able to use sublime as an IDE it has 1. Tree based dir 1. Search/replace via regex 1. Sidebar with useful info 1. Git plugins, I mostly used git diff & git blame 1. Sort imports, linting, syntax highlighting etc.

And then I used terminal for 1. Code formatting 1. Server 1. Git based commands for commit, push, rebase etc.

Why would one use sublime?

The biggest advantage was that the same setup would work for Coffeescript, Typesript, python, Java, lisp, C++ and many others.

This setup was so much easy to follow and thus I used it for almost 4 years. This setup also made me more concious programmer and I was aware of things I needed for effective coding.

What about Atom?

An alternative of this is Atom which is another powerful text editor which many use as their personalized IDE.

Intellij

I started using it when I was going to write most of my code in Java/Scala.

There are only few things I can say in its favor. Intellij is one of the best IDEs out there with most features. In their latest release of Intellij 2020.2 they even had auto-import for implicits.

They have the most seamless build imports and then one would get all the features for a language.

I would recommend one to use Intellij when learning a new language. Intellij recommends idiomatic way of writing the same code in a language. To use a new language in Intellij one can install the intellij’s language plugin for it and would get all the features.

Awesome! Isn’t it?

VIM

Having used Intellij for almost a year, I wanted to try VIM.

Motivation : I wanted to try digitizing my notes and was looking at various text editors and how others do it. Vim was a good choice.

Vim was the 5th most popular IDE in stackoverflow 2019 survey. So I decided to give it a try.

My setup- 1. Vim has language support via plugins. 1. I used plugins for autocomplete, fuzzysearch, tags, etc. 1. I build my scala project using metals as a LSP. 1. I had all the IDE features available in VIM.

Other than the project 1. I was able to use VIM effectively for digital note taking. 1. I used markdown.

Disadvantage

VIM took time getting used to. But felt like it was worth it. I would certainly not learn a new langauge in VIM but then move to VIM after I am comfortable with it.

Is VIM an ide?

VIM is a good text editor, and no one had any opinion regarding a text editor before. Its not an IDE, but one can configure VIM for the features one mostly uses in an IDE.

Why use it?

I personally felt increase in productivity since I was now getting more aware as a developer. Whenever I thought I needed a new feature I had to think hard about what I want to do with it and then I would use it effectively since I spent time thinking about the feature and its uses for myself.

Emacs

Recently after using VIM for almost 6 months, I wanted to try emacs but was afraid to use it based on reviews I read which mentions that emacs is tought to get onboarded with.

Then comes the emacs-evil.

Once I was used to basic navigation and workflow in vim it took me almost 1 week to transition to emacs.

Well, why not?

Digital note taking moved to org-mode, which I think is more powerful and seems like Evernote to me.

Is it better than vim?

I had same setup with VIM and it works almost the same. I was able to set it up much easily and use it for projects within a week.

My thoughts

I do not think every VIM user should move to emacs but emacs isn’t bad either. Org-mode lured me to use emacs and then I found language setup is also seemless in emacs. The easy setup in emacs (almost like Intellij) was something I liked.

Is my journey over?

No

I am planning to try the most popular IDE based on stackoverflow 2019 survey - Visual Studio. Right now I am still experimenting with emacs and then will setup my workflow before trying out VS next year.

Conclusion

My workflow isn’t the best and it doesn’t have everything, but certainly it has everything I need to be efficient and productive.

I do not want to stick to one particular tool (an IDE) and keep trying different workflows to understand what I lack and be able to include it in my workflow.

Some of the IDEs I have tried have been text editors which can be used as IDE by use of plugins which provides best flexibility.

My advice would be for one to figure out the features they mostly use and also keep looking at other tools for the features that they are missing.

Keep coding!
